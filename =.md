# UmrahHaji.com UX Design Research Recommendation

Status: Recommended UX Direction  
Date: 29 June 2026  
Input: Attached design.md suggestion, JUV Homepage PRD, current JUV Homepage design guide  
Primary UI Reference: muslim101.id  

---

## 1. Executive Decision

Use the attached `design.md` suggestion as the **global design standard** for UmrahHaji.com, but do not adopt it blindly.

Best UX solution:

```text
Modern Pilgrimage Marketplace + Journey Management
```

This means the product should feel like:

1. A trusted Umrah/Hajj travel marketplace.
2. A comparison tool for packages, agencies, hotels, flights, dates, and prices.
3. A safe booking and payment journey.
4. A document and trip-status tracking system.
5. A spiritual travel experience that feels calm and respectful.

It should not feel like:

1. Generic SaaS landing page.
2. Generic travel agency website.
3. Luxury hotel brochure.
4. Decorative Islamic event website.
5. Admin dashboard.

## 2. Research-Based UX Principles

### 2.1 Search First, But Do Not Overload Homepage Filters

Homepage should make package search immediately visible.

Recommended homepage search fields:

1. Package type.
2. Departure city.
3. Travel date/month.
4. Number of pilgrims.
5. Room preference or budget.

Advanced filters should move to search results:

1. Hotel class.
2. Distance to mosque.
3. Airline.
4. Flight type.
5. Meals.
6. Visa included.
7. Mutawwif included.
8. Payment option.
9. Agency rating.

Reason:

1. Homepage is for entry and intent capture.
2. Search results are for comparison and refinement.
3. Mobile users should not face a long filter form before seeing packages.

UX decision:

```text
Homepage = quick search + category chips.
Search results = full faceted filtering.
```

---

### 2.2 Package Cards Must Be Decision Tools

Package cards are the core conversion component.

A good package card should help users decide:

1. Is this package relevant?
2. Is the agency trustworthy?
3. Is the price clear?
4. Is the schedule suitable?
5. Is the hotel/flight quality acceptable?
6. What should I do next?

Recommended card hierarchy:

| Priority | Content |
| --- | --- |
| Primary | Package name, promo/final price, departure month, duration, agency name, verified badge |
| Secondary | Departure city, hotel class, hotel distance, airline/flight, room type |
| Tertiary | Rating, seats left, payment note, favorite/compare action |

Do not make package cards look like generic holiday cards with only image, title, and price.

---

### 2.3 Trust Must Be Contextual

Trust should appear where users make risky decisions.

Place trust signals in:

1. Package card: verified agency badge.
2. Package detail: agency license, included/not included, cancellation/payment terms.
3. Booking flow: booking summary, deposit/balance clarity.
4. Payment flow: payment status, confirmation, next step.
5. My Trip: document and visa status.
6. Support: visible help channel and expected response.

Avoid generic trust blocks:

1. Random logos.
2. Unverifiable awards.
3. Fake countdowns.
4. Unsupported "trusted by thousands" claims.
5. Icon cards without data.

UX decision:

```text
Trust is not a marketing section only. It must be embedded inside package, booking, payment, and trip status UI.
```

---

### 2.4 Use Progressive Disclosure for Complex Travel Details

Umrah/Hajj packages carry many details:

1. Hotel distance.
2. Room sharing.
3. Visa.
4. Mutawwif.
5. Meals.
6. Transport.
7. Payment terms.
8. Cancellation policy.
9. Document requirements.

Show them in layers:

1. Summary card.
2. Comparison row.
3. Detail tabs/accordions.
4. Policy section.
5. Downloadable/printable confirmation where needed.

Do not hide:

1. Price conditions.
2. Cancellation policy.
3. Required documents.
4. Payment due dates.
5. Package exclusions.

UX decision:

```text
Use progressive disclosure for detail density, not for hiding risk.
```

---

### 2.5 Mobile Must Be Marketplace-First

Most users may arrive from WhatsApp, social links, search engines, or agency-shared package links.

Mobile priority:

1. Search package.
2. Browse promo/category.
3. Compare key package facts.
4. Check agency trust.
5. Continue booking.
6. Track My Trip.
7. Access Guides and Profile.

Recommended bottom navigation:

```text
Home, Packages, My Trip, Guides, Profile
```

Use `Guides` as customer-facing label. Keep `Guidelist` only as an internal/module term if needed.

---

### 2.6 Booking and Payment Need Error Prevention

Booking and payment create financial commitment, so UX must include:

1. Review step before confirmation.
2. Clear total price.
3. Deposit and remaining balance.
4. Payment due date.
5. Included/not included.
6. Cancellation/refund policy link.
7. Editable traveler details before final submit.
8. Clear success and pending states.

UX decision:

```text
Every financial action must allow review, confirmation, and correction before final submission.
```

---

### 2.7 Accessibility Is a Core Product Requirement

Umrah/Hajj users can include elderly pilgrims and families booking on behalf of parents.

Minimum UX requirements:

1. Body text at least 15-16 px.
2. Comfortable touch targets, preferably 44 x 44 px.
3. No color-only status indicators.
4. Visible labels for inputs.
5. Clear focus states.
6. Useful error messages.
7. Reduced-motion support.
8. Avoid tiny grey metadata.
9. Avoid low-contrast gold text.
10. Support keyboard and screen-reader behavior.

UX decision:

```text
Design for older and lower-confidence digital users first.
```

## 4. What To Adjust

### 4.1 Color System

The attached file suggests a deep green/gold palette. For this project, override with the approved brand colors:

| Token | Hex | Usage |
| --- | --- | --- |
| Primary | `#0694A2` | Main CTA, active states, search button, selected tabs |
| Secondary | `#C27803` | Promo, discount, premium/highlight accent |
| Primary Soft | `#E6F7F9` | Soft background, selected chip background |
| Secondary Soft | `#FFF3DC` | Promo background, offer tint |
| Text Primary | `#172326` | Main text |
| Text Secondary | `#5D6B70` | Metadata |
| Border | `#DDE7EA` | Dividers, card borders |
| Surface | `#FFFFFF` | Cards, forms, search panel |
| Page Background | `#F7FAFA` | Main page background |

Rules:

1. Primary `#0694A2` owns conversion actions.
2. Secondary `#C27803` owns promo/commercial emphasis.
3. Do not make the entire UI teal/gold.
4. Use neutral surfaces for dense package data.

---

### 4.2 Currency and Market Copy

The product context is English copy with RM currency.

Use:

```text
RM 7,890
per pilgrim
```

Do not use:

```text
Rp 31.500.000
per jamaah
```

Formatting rules:

1. Use English customer-facing copy.
2. Use RM for package, deposit, balance, cancellation, and payment amounts.
3. Use `pilgrim` or `traveller` in customer-facing copy.
4. Keep `Jamaah/User View` only as role/module naming if existing PRDs already use it.
5. Do not mix English headings with Indonesian CTA labels.

---

### 4.3 Labels

Use customer-facing labels:

| Avoid | Use |
| --- | --- |
| Guidelist | Guides |
| Get Started | Search Packages |
| Discover | View Packages |
| Pax | pilgrim |
| Jamaah | pilgrim, user, or traveller |
| Travel Dashboard | My Trip |

## 5. Best Homepage UX Structure

Recommended homepage rhythm:

1. Top navbar.
2. Search-first hero.
3. Category chips: Umrah, Hajj, Muslim Trip, Promo, Family.
4. Limited offers.
5. Featured package rows.
6. Why compare on UmrahHaji.com.
7. Verified agency trust block.
8. Booking and payment safety block.
9. Destination inspiration.
10. Pilgrim testimonials.
11. Guides.
12. FAQ.
13. Footer.

Every section must support one of these:

```text
Search, compare, trust, learn, book.
```

If a section does not support one of those, remove it.

## 6. Best Search Results UX

Search results should be the main comparison workspace.

Recommended layout:

### Mobile

1. Sticky search summary.
2. Sort/filter button.
3. Active filter chips.
4. Package list.
5. Filter bottom sheet.
6. Compare/save actions.

### Desktop

1. Search summary bar.
2. Left filter rail.
3. Result count and sort.
4. Package cards/list.
5. Optional compare tray.

Recommended filters:

1. Package type.
2. Departure month.
3. Departure city.
4. Price range.
5. Duration.
6. Hotel class.
7. Distance to Masjid.
8. Airline.
9. Room arrangement.
10. Agency rating.
11. Promo only.
12. Available seats.

## 7. Best Booking Flow UX

Recommended booking steps:

```text
Package Detail
-> Select date/room/travelers
-> Traveler information
-> Document requirements preview
-> Payment option / deposit
-> Review booking
-> Confirm booking
-> Booking status
```

Rules:

1. Keep booking summary sticky on desktop.
2. Keep total price visible.
3. Explain deposit and remaining balance.
4. Show required documents before payment where relevant.
5. Do not force account creation too early if product allows guest browsing.
6. Login/register gate should appear at booking commitment point, not at package discovery.

---

## 8. Best My Trip UX

My Trip should reassure users after booking.

Recommended structure:

1. Trip status summary.
2. Next required action.
3. Document checklist.
4. Payment status.
5. Departure readiness.
6. Mutawwif/group information.
7. Important announcements.
8. Support/contact.

Use a timeline only if each stage has useful information:

1. Status.
2. Date.
3. Required action.
4. Responsible party.
5. Explanation.

## 10. Updated UX QA Checklist

Before approving any screen:

1. Can users understand the primary action in 5 seconds?
2. Is the screen specific to Umrah/Hajj, not generic travel?
3. Is agency verification visible where trust is needed?
4. Are package price, date, duration, and hotel quality clear?
5. Is the CTA specific and action-oriented?
6. Are important policies accessible?
7. Are errors specific and recoverable?
8. Are touch targets comfortable?
9. Are states designed: loading, empty, error, success?
10. Does the screen work well on mobile first?
11. Does it avoid fake urgency and unsupported claims?
12. Does it support older users and family decision-makers?

## 12. Final Recommendation

Adopt the attached design suggestion as the global standard, with these final decisions:

1. Keep the approved primary color `#0694A2`.
2. Keep the approved secondary color `#C27803`.
3. Use `Modern Pilgrimage Marketplace + Journey Management` as the UX direction.
4. Treat package cards, agency trust, booking status, payment clarity, and document tracking as the core product identity.
5. Use Muslim101 only as marketplace reference, not as a visual copy target.
6. Design mobile first for English-speaking pilgrims and family decision-makers using RM pricing.
7. Reject any UI that feels generic, decorative-only, or not useful for search, comparison, trust, learning, or booking.
